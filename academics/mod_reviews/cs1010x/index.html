CS1010X

December 2, 2022

I have cleared this module under iBLOCS, which is a programme catered to those who have ORDed or are ORDing soon. At the end of this 6 month course, I cleared my CS1010E requirement and earned a letter grade which can be transferred over to my CAP.

Content#

This module taught some basic programming concepts, such as scoping, loops and functions, using mostly Python and a bit of Java.

  1. Introduction to CS1010X & Basics of Python
  2. Functional Abstraction
  3. Good Abstraction and Introduction to Recursion
  4. Recursion, Iteration + Order of Growth
  5. Debugging
  6. Higher Order Function
  7. Data Abstraction
  8. Working with Sequences
  9. Searching and Sorting
  10. Multiple Representations
  11. Generic Operators
  12. Object Oriented Programming
  13. Dynamic Programming & Memoization, Exception Handling
  14. Introduction to Java Programming
  15. Object Oriented Programming with Java

The content seems more difficult/heavy than the usual CS1010 courses, but we do have more time to learn too. The Prof also sometimes goes into deeper concepts. You probably do not need to learn Python before hand, but knowing some of syntax can help you get up to pace faster. Anyway, here are some key takeaways.

The Awakening Moment#

Some of the concepts (e.g. recursion) are quite abstract. I spent quite a lot of time thinking of how to solve some of the missions (assignments); some can even take up weeks. But after solving them, I think it is good to reflect on the thinking process. After a while, I got a hang of it.

Not Exactly a Python Course#

The Prof did explain that this course is not a course that teaches programming in Python. Instead, Python is used to teach programming concepts. I have to agree with this. It is good to go in with a mindset that you are not learning a specific programming language as it reduces the mental barrier when trying a new language.

Useful Outside CS#

Although this is a CS coded module, some of the concepts are useful outside of CS. When I took EG1311, I actually encountered some of the concepts being used to programme the Arduino. Some modules from other departments may also include plenty of coding.

Assessment#

  • Coursemology 33%
  • Participation 2%
  • Midterm 16%
  • Practical Exam 16%
  • Finals 33%

Almost everyone got the full 35% for coursemology and participation. For the exams, it is really important to understand the concepts, as the style of the problems changed from one batch before me. It is no longer the typical reading the CSV file and short problems. The questions tested more computational thinking. One of the questions does not even involve coding (I think I saw it in some Math Olympiad before). You can’t do all the past year papers and expect to be able to be prepared for the exam.

General#

I really enjoyed this module. Having six months to study for one module also makes this module generally manageable, even when I am working part time. Learning programming through a module also lays out the concepts in a more systematic manner to be learnt. I would recommend people to take it.

Final Grade: A
